Redesign itemku lite Apps

UI/UX Portfolio

Background Company

itemku is a customer-type bold shopping site for customers that focuses on virtual product transactions such as items, accounts and video game currencies as well as various types of vouchers. itemku Founded in 2014 in Jakarta as a subsidiary of PT Five Jack, a startup company from South Korea.

Pain Point

This Pain Point was created based on my assumptions as a user of the itemku application.

  1. Users find it difficult to search for a product or item on the home menu because there is no search on that page.
  2. Users feel charged when opening the “fast shopping” page. Users are directed to access it via a web browser (excluding apps).
  3. Users find it difficult to contact the seller before the user buys the product/item they choose.
  4. Users feel burdened when they want to buy a product, they cannot buy it directly, in other words they have to add it to the basket first

Room for Improvement

From the paint points found to outline the improvements.

  1. Users want to search for the product or item they want without having to do it manually.
  2. Users want to carry out activities completely within the app without having to intervene with a web browser.
  3. Users want to ask questions and want to make sure first about the product or item they have chosen because they cannot contact the seller before buying.
  4. When the user finds the product or item they are looking for, they want to be able to immediately buy the product without having to carry out repeated orders.

Design Mandates

After gaining some insight, then create a Design Mandates that can be used as a reference for making improvements.

  1. As a user, i can get the product search feature in the application.
  2. As a user, I can access “fast shopping” in the application.
  3. As a user, I can make contact between the user and the seller before making a transaction.
  4. As a user, I can make direct purchases of the selected products.
